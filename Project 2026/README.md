## Project Description

Space Colony is an Android application where space colony crew is managed by the player. The program features five different crew member specializations which are: **Pilot, Engineer, Medic, Scientist and Soldier**. The user can recruit different crew members, train them in a simulator to gain experience points and send them on cooperative turn-based missions against threats generated by the system.

The application is built using **Java** in **Android Studio** and it follows **Object-Oriented Programming** principles, which includes inheritance, encapsulation and polymorphism. The app features a dark space-themed UI with colored crew cards, live energy bars and a tactical combat system.

## Implemented Features

### Mandatory Features
- **Object-Oriented Programming**: The program uses inheritance (CrewMember → Pilot/Engineer/Medic/Scientist/Soldier), encapsulation (private fields with getters/setters) and polymorphism (each subclass overrides `getSpecialBonus()` and `actWithBonus()`).
- **All code in English**: All Java code, comments and documentation are written in English.
- **Android Compatibility**: The app runs on Android devices (minSdk 28, API 28+) and is developed in Java using Android Studio.
- **Crew Management**: Users can create crew members of 5 different specializations. New crew members are placed in Quarters. Users can move crew members between Quarters, Simulator and Mission Control.
- **Training System**: Crew members gain +1 experience point per training session in the Simulator. Experience permanently increases their effective skill power.
- **Cooperative Mission System**: Two crew members are selected for each mission. A threat is generated by the system with stats that scale based on mission count. Missions are based on turns with tactical combat choices. Surviving crew members earn experience points. Defeated crew members are removed from the program.
- **Crew Recovery**: When a crew member returns to Quarters, their energy is fully restored but experience points are kept.
- **Data Structures**: HashMap<Integer, CrewMember> is used in Storage for efficient crew member lookup by ID. ArrayList is used for filtered crew lists in RecyclerView adapters.

## Application Use-Flow

1. **App Launch** → MainActivity loads saved data and shows colony status
2. **Recruit** → User enters name, selects specialization then, crew member added to Quarters
3. **Quarters** → User selects crew members and then moves them to Simulator or Mission Control
4. **Simulator** → User selects crew members after that, trains them (+1 XP each) then sends back to Quarters (energy gets restored)
5. **Mission Control** → User taps two crew members to select them and launches mission
6. **Mission** → Turn-based combat: player chooses Attack/Defend/Special then threat retaliates after that it repeats until someone runs out of energy
7. **Mission Complete** → Surviving crew gain XP and returned to Mission Control
8. **Mission Failed** → Both crew members removed from the game permanently
9. **Statistics** → View bar chart of crew performance and individual stats
10. **Data is saved automatically** after every action

## Installation Instructions

1. Clone the repository:
   ```
   git clone https://github.com/Yasir47610/Space_Colony.git
   ```
2. Open **Android Studio**
3. Select **File then Open** and navigate to the `Project 2026` folder
4. Wait for Gradle sync to complete
5. Connect an Android device (API 28+) with USB Debugging enabled, or use an emulator
6. Press the **Run** button

**Requirements:**
- Android Studio Hedgehog or newer
- Android SDK API 28 or higher
- Java 11
